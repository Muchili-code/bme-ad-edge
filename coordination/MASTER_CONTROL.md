# 母进程总控说明

## 1. 母进程职责

母进程负责统筹整个边缘端 Demo 的多进程协作，不直接把所有代码一次性写完。母进程的主要职责是：

1. 维护共享契约文件。
2. 分配模块和任务给子进程。
3. 检查各模块是否按契约实现。
4. 处理跨模块冲突。
5. 更新 `IMPLEMENTATION_STATUS.md`。
6. 执行最终验收和反虚跑测试。

## 2. 推荐执行层次

```text
第 0 层：契约与项目骨架
第 1 层：mock 病例包与数据校验
第 2 层：FastAPI 后端与 DTI/V1/RI 计算
第 3 层：React 前端与展示流程
第 4 层：WSL 部署、一键启动与验收测试
```

## 3. 串并结合策略

必须串行完成：

1. 共享契约确认。
2. mock 病例包结构确认。
3. 后端核心 API 与输出文件格式确认。
4. 前端联调和最终验收。

可以并行推进：

1. 前端静态页面骨架与后端 API 骨架。
2. mock 数据生成脚本与病例包校验脚本。
3. 部署文档草稿与 WSL 启动脚本草稿。

## 4. 文件写入边界

| 角色 | 主要写入范围 |
| :--- | :--- |
| 母进程 | `coordination/`、总状态、跨模块整合、最终验收记录 |
| 数据模块 | `scripts/`、`data/cases/`、数据相关测试 |
| 后端模块 | `backend/`、后端测试 |
| 前端模块 | `frontend/`、前端测试 |
| 部署模块 | `start_demo.sh`、`.env.example`、`docs/` |

共享契约文件默认只由母进程修改。子进程发现契约问题时，写入自己的状态文件，不直接改契约。

## 5. 母进程检查清单

每轮检查至少确认：

1. 子进程是否只改了自己的写入范围。
2. API 是否符合 `API_CONTRACT.md`。
3. 病例包是否符合 `CASE_PACKAGE_SPEC.md`。
4. 输出文件是否符合 `OUTPUT_FILE_SPEC.md`。
5. 页面文案是否避免错误声称边缘端真实运行完整 DisC-Diff。
6. 报告是否来自后端生成文件。
7. WSL 环境是否没有混用 Windows Node/Python。
