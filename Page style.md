# 边缘计算端 App 前端界面风格设计规范 (Page Style Guide)

## 1. 总体视觉定位
本项目（阿尔茨海默病 dMRI 边缘计算辅助分析“黑盒”）的前端界面定位为**“现代高级医疗数据看板 (Premium Medical Dashboard)”**。
核心特色在于提供**“深浅双色主题无缝切换 (Light & Dark Mode Toggle)”**功能。为了适应比赛答辩的大屏投影展示以及医疗现场终端触摸屏的不同光线环境，前端界面在整体布局排版保持高度一致的前提下，支持通过点击或触控屏幕的“主题切换”按钮，一键无缝转换深/浅色模式。

## 2. 界面布局与交互结构 (Layout Structure)
无论当前是深色还是浅色模式，系统均采用以下统一、清晰的结构：
- **全局布局**：大屏卡片式模块化排版（参考现代 Pad 端或大屏工作站应用）。将不同内容区通过清晰的卡片容器分离，保证信息层级一目了然。
- **字体排印 (Typography)**：全面弃用浏览器默认字体，采用现代无衬线字体体系（务必确认支持中英文结合），确保数值、报表与英文字段显得极简、专业且具高级科技感。

## 3. 深色模式 (Dark Mode) —— 赛博医疗与边缘硬核算力感
*场景定位：适用于硬件盒子本身自带的小型触控屏，或在暗光环境下向评委展示高科技的底层边缘算力。*

- **背景与底色**：深炭灰或极夜蓝（例如 `#121212` 或 `#1A1A24`），这种色彩不仅减少屏幕眩光，还能使本身黑底的 MRI 影像完美融入背景画布，消除突兀的边界感。
- **UI 材质**：采用**毛玻璃特效 (Glassmorphism)**。半透明的深色卡片结合轻微的高斯模糊，边缘辅以 `1px` 的半透明白线勾勒，营造强烈的空间层级与未来科技感。
- **核心数据高亮 (Accents)**：
  - **青色 (Cyan/Teal)**：用于指示正常的生命体征、安全的 ROI 脑区以及 AI 处理进度。
  - **深空紫/霓虹紫 (Purple Neon)**：用于标注神经学异常数据，对于需要警示的高风险 RI 曲线（如 Skewness 偏高），点缀以霓虹发光效果（Glow），凸显极客气质。

## 4. 浅色模式 (Light Mode) —— 临床无菌感与成熟商用级产品
*场景定位：适用于比赛现场在明亮的幻灯片或大投影仪上展示，传达这套系统是“一款能够真正落地进入三甲医院使用的成熟医疗软件”。*

- **背景与底色**：极浅的冷灰白（例如 `#F8F9FA`）。背景洁净，文字全部使用高对比度的深石板灰（`#333333`），确保在远距离投影时拥有最高级别的可读性和清晰度。
- **UI 材质**：采用**柔和投影特效 (Neumorphism / Soft Drop Shadows)**。纯白色的模块卡片仿佛悬浮在浅色背景之上。由于卡片是纯白色，黑色的 MRI 影像放入其中会像“装在相框里”一样被强烈凸显，显得非常无菌、专业、严谨。
- **核心数据高亮 (Accents)**：
  - **经典医疗蓝 (Medical Blue, #0056D2)**：作为全局主要控件和操作按钮的主色调，赋予系统令人安心的权威医疗属性。
  - **薄荷绿与预警橘红**：图表区正常的皮层柱指标用浅绿色标注；而存在微结构退化风险（如 RI Maximum 显著降低的额叶区）则采用柔和的橘红/红色预警，保持临床诊断的严肃性。

## 5. 交互与动画要求
- 前端需在顶级导航栏或全局醒目位置设置 **Theme Toggle 控制开关（月亮/太阳图标）**。
- **平滑过渡**：深浅模式切换时必须引入 CSS 渐变动画（约 `0.3s` 过渡时间），不允许出现颜色生硬闪烁或页面卡顿。
- **全生态响应**：一键切换时，不仅仅是改变背景色，图表（ECharts）内部的坐标轴网格、文字说明、数据线颜色也必须同时响应并反转配色，确保无论哪种模式下，图表的可读性都能保持完美。
